# TH2W / TH2.VIP 接口语义文档

## 1. 文档定位

本文不是传统 API 手册，而是接口语义文档。它只描述其他文档中已经提到的接口能力，用来帮助人类和 AI 理解“系统有哪些交互边界、谁调用、完成什么业务、输入输出大致是什么”。

本文不列出具体路径、HTTP 方法、curl 示例、请求头、字段明细或完整响应结构。

## 2. 接口总体风格

已确认事实：

- 玩家和 APP / H5 侧接口通常返回业务 code、message 和 data 语义。
- 登录态主要通过会员 API token 表达。
- 后台接口由 Dcat Admin 登录态和权限能力保护。
- 内部实时客服支持游客 visitor id 和可选会员 token 两种身份上下文。
- 第三方回调接口具有独立的响应语义，尤其是 WXGame。
- 新旧接口并存，存在兼容入口。

合理推断：

- 前台调用方包括桌面静态页、手机 H5、旧 APP 和活动脚本。
- 后台调用方包括 Dcat Admin 页面、TCG 风格页面和运营人员操作。
- 第三方调用方包括游戏厂商、支付渠道和聊天服务。

## 3. 用户与身份接口

### 【接口：用户注册与登录】

调用方：前端请求。

功能说明：

用于游客创建账号或会员登录。注册支持邀请码，登录成功后返回会员身份和访问凭证，前端保存凭证并进入会员中心、游戏启动页或原 redirect 目标。

入参概述：

账号凭证、密码、注册确认信息、邀请码、客户端上下文。

输出概述：

登录或注册结果、会员基本身份、访问凭证、提示信息。

### 【接口：用户资料与余额】

调用方：前端请求。

功能说明：

用于会员中心加载用户名、余额、VIP 等级、基础资料和当前账户状态。前台资金工具、钱包页和游戏入口都会依赖该能力。

入参概述：

会员访问凭证和必要的查询上下文。

输出概述：

会员资料、主账户余额、状态信息和可展示的会员属性。

### 【接口：登录密码和支付密码维护】

调用方：前端请求。

功能说明：

用于会员修改登录密码或取款密码。取款密码参与提现校验。

入参概述：

会员身份、旧密码或校验信息、新密码。

输出概述：

修改结果和失败原因。

### 【接口：会员注销或退出】

调用方：前端请求。

功能说明：

用于会员结束当前登录态或清理访问凭证。前端还需要同步清理本地 token。

入参概述：

会员访问凭证。

输出概述：

退出结果。

## 4. 游戏接口

### 【接口：游戏目录查询】

调用方：前端请求。

功能说明：

用于首页、游戏大厅、分类页和移动端加载可展示游戏。游戏目录受游戏平台、游戏分类、热门、新上架、推荐、排序、PC / APP 状态和站点状态影响。

入参概述：

平台、游戏类型、关键词、分类、终端上下文。

输出概述：

游戏列表、平台信息、展示图片、排序信息和状态信息。

### 【接口：游戏启动】

调用方：前端请求。

功能说明：

用于会员进入具体游戏。后端会校验用户、平台、游戏状态和运营限制，并按平台模式调用第三方游戏网关或 WXGame 获取游戏入口。需要时会执行主余额和游戏平台余额之间的自动转入或回收。

入参概述：

会员身份、游戏平台、游戏类型、游戏标识、语言和终端上下文。

输出概述：

游戏入口、启动结果、失败原因或余额状态。

### 【接口：游戏余额转入转出】

调用方：前端请求。

功能说明：

用于主钱包和游戏平台余额之间的手动转换，也用于一键回收。新资金链路使用安全转账服务，包含 pending 流水、外部调用状态、本地落账状态和恢复状态。

入参概述：

会员身份、平台、金额、转账方向。

输出概述：

转账结果、余额、处理中或失败原因。

### 【接口：游戏投注记录查询】

调用方：前端请求。

功能说明：

用于会员或代理查看投注记录、有效投注、输赢和游戏流水。后台运营也使用游戏记录进行报表和审计。

入参概述：

会员或团队上下文、时间范围、平台、游戏类型、分页条件。

输出概述：

投注记录列表、金额统计和分页信息。

### 【接口：第三方游戏网关调用】

调用方：后端调用。

功能说明：

后端通过游戏服务向第三方聚合网关发起注册、登录、余额查询、转入、转出、额度查询、游戏列表和记录查询。

入参概述：

商户配置、会员账号、平台、金额、订单号、游戏参数。

输出概述：

第三方返回状态、游戏入口、余额或错误信息。

### 【接口：WXGame 玩家校验】

调用方：后端调用 / 第三方回调。

功能说明：

WXGame 根据玩家 token 校验玩家身份，并获取玩家标识、余额和币种。后端会校验 token 签名、用户状态和可选回调签名。

入参概述：

玩家 token、回调签名上下文。

输出概述：

玩家身份、余额、币种或拒绝原因。

### 【接口：WXGame 余额查询】

调用方：第三方回调。

功能说明：

WXGame 在游戏过程中查询玩家主账户余额。后端按玩家标识查找用户并返回余额。

入参概述：

玩家标识、币种和回调签名上下文。

输出概述：

余额、币种或错误原因。

### 【接口：WXGame 下注】

调用方：第三方回调。

功能说明：

WXGame 发起下注扣款。后端校验签名、币种、玩家、交易幂等、游戏限制和玩家限额，然后在事务中扣减余额并写入转账流水和游戏记录。

入参概述：

玩家标识、交易标识、投注金额、游戏品牌、游戏类型、游戏标识和币种。

输出概述：

扣款结果、最新余额或错误原因。

### 【接口：WXGame 派奖】

调用方：第三方回调。

功能说明：

WXGame 发起中奖派发。后端校验交易幂等和关联下注，成功后增加余额并写入流水和游戏记录。

入参概述：

玩家标识、交易标识、派彩金额、关联下注、游戏上下文和币种。

输出概述：

派奖结果、最新余额或错误原因。

### 【接口：WXGame 退款】

调用方：第三方回调。

功能说明：

WXGame 发起退款。后端校验关联下注和交易幂等，成功后回补余额并记录退款流水。

入参概述：

玩家标识、交易标识、退款金额、关联下注、游戏上下文和币种。

输出概述：

退款结果、最新余额或错误原因。

### 【接口：WXGame 状态查询】

调用方：后端调用 / 管理检查。

功能说明：

用于查看 WXGame 是否启用、配置是否完整、回调域名、币种和签名要求。

入参概述：

管理或系统查询上下文。

输出概述：

WXGame 启用状态、配置状态、回调语义和币种信息。

## 5. 资金接口

### 【接口：充值申请】

调用方：前端请求。

功能说明：

用于会员提交充值。系统根据支付方式、金额、通道范围、商户配置和用户状态创建充值单，并返回支付入口或线下入款信息。

入参概述：

会员身份、充值金额、支付方式、支付通道和客户端上下文。

输出概述：

充值单结果、支付入口、入款资料或失败原因。

### 【接口：充值通道与范围查询】

调用方：前端请求。

功能说明：

用于前台展示可用支付方式、支付账号、金额上下限和渠道说明。

入参概述：

会员身份、支付类型或终端上下文。

输出概述：

支付方式列表、通道状态、金额范围和展示说明。

### 【接口：支付回调处理】

调用方：第三方回调。

功能说明：

用于支付渠道通知充值状态。后端根据订单、签名或通道规则更新充值单和用户余额。

入参概述：

渠道订单、平台订单、支付状态、金额、签名或渠道上下文。

输出概述：

处理结果、订单状态或拒绝原因。

### 【接口：提现申请】

调用方：前端请求。

功能说明：

用于会员提交提现。系统校验余额、提现时间、次数、金额范围、打码量、收款账户和支付密码，通过后创建提现单并预扣余额。

入参概述：

会员身份、提现金额、收款账户、取款密码和提现类型。

输出概述：

提现申请结果、审核状态、余额变化或失败原因。

### 【接口：银行卡与虚拟币地址管理】

调用方：前端请求。

功能说明：

用于会员绑定、查询或删除银行卡、USDT、EBPay 等收款资料，供提现流程使用。

入参概述：

会员身份、账户类型、收款账户资料和操作类型。

输出概述：

收款资料列表、绑定结果或删除结果。

### 【接口：充值提现记录查询】

调用方：前端请求。

功能说明：

用于会员中心展示近期充值、提现和资金历史。代理也可以查询下级资金记录。

入参概述：

会员或团队上下文、记录类型、时间范围、分页条件。

输出概述：

资金记录、状态、金额和分页信息。

### 【接口：返水领取】

调用方：前端请求。

功能说明：

用于会员查询和领取返水。返水开关和最低返水金额来自系统配置。

入参概述：

会员身份和查询上下文。

输出概述：

可领取状态、领取结果或失败原因。

### 【接口：红包领取】

调用方：前端请求。

功能说明：

用于会员领取红包或查询红包记录。红包开关由系统配置控制。

入参概述：

会员身份、红包上下文。

输出概述：

领取结果、红包金额或失败原因。

## 6. 活动与推广接口

### 【接口：活动分类与活动列表】

调用方：前端请求。

功能说明：

用于活动页展示分类和活动卡片。后端按启用状态、渠道、时间窗和排序过滤，并优先返回前台本地化展示名，同时保留后台管理名语义。

入参概述：

渠道、分类、终端、分页或筛选上下文。

输出概述：

活动分类、活动列表、前台展示文案、后台管理名、展示图片、排序和活动状态。

### 【接口：活动详情与弹窗】

调用方：前端请求。

功能说明：

用于展示单个活动详情和首页弹窗活动。弹窗受活动配置、渠道和前端频率控制共同影响。

入参概述：

活动标识、渠道、终端和弹窗上下文。

输出概述：

活动详情、弹窗图片、行动按钮、活动规则和可申请状态。

### 【接口：活动曝光记录】

调用方：前端请求。

功能说明：

用于记录活动详情或首页弹窗曝光，帮助运营分析活动展示效果。

入参概述：

活动标识、曝光来源、会话上下文和终端信息。

输出概述：

记录结果。

### 【接口：活动申请】

调用方：前端请求。

功能说明：

用于会员申请活动。后端会检查登录态、活动可见性、是否允许申请、重复申请、黑名单和活动券。活动翻倍规则已具备规则选择能力，但当前接口语义文档不把它描述为已完成发奖闭环。

入参概述：

会员身份、活动标识、活动券或申请上下文。

输出概述：

申请结果、审核状态或限制原因。

### 【接口：推广渠道资料维护】

调用方：后台接口。

功能说明：

用于后台维护推广链接、推广域名、落地页、SEO、推送和事件配置。

入参概述：

后台用户身份、页面 code、推广配置、状态和排序信息。

输出概述：

保存结果、列表数据、导出结果或失败原因。

## 7. 客服与消息接口

### 【接口：客服配置与入口】

调用方：前端请求。

功能说明：

用于前台获取客服入口。系统会根据外部链接、平台客服表、外部实时客服、内部实时客服、Stream Chat、工单开关和玩家等级组合返回可用客服方式。

入参概述：

会员身份、玩家等级、终端上下文。

输出概述：

客服模式、实时客服提供方、客服入口、联系方式、工单入口、内部实时客服状态和聊天配置状态。

### 【接口：实时客服会话】

调用方：前端请求。

功能说明：

用于游客或会员建立、复用内部实时客服会话。游客依赖浏览器侧 visitor id，会员可以通过登录凭证把会话绑定到账号。系统会检查内部实时客服开关和必要数据表是否可用。

入参概述：

访客上下文、可选会员身份和可选会话上下文。

输出概述：

会话状态、已有消息、轮询建议和不可用原因。

### 【接口：实时客服消息轮询】

调用方：前端请求。

功能说明：

用于前台按会话拉取新增消息，并把后台客服消息标记为用户侧已读。

入参概述：

访客或会员上下文、会话上下文和已接收消息位置。

输出概述：

会话状态、新增消息列表和未读状态。

### 【接口：实时客服消息发送】

调用方：前端请求。

功能说明：

用于游客或会员向内部客服会话发送文本消息。系统会复用或创建打开中的会话，更新最后消息和后台未读数。

入参概述：

访客或会员上下文、会话上下文和消息内容。

输出概述：

发送结果、消息记录、会话状态和失败原因。

### 【接口：实时客服会话关闭】

调用方：前端请求。

功能说明：

用于前台关闭自己的内部实时客服会话。关闭后再次咨询会创建或复用新的打开会话。

入参概述：

访客或会员上下文和会话上下文。

输出概述：

关闭结果和会话状态。

### 【接口：工单创建与查询】

调用方：前端请求。

功能说明：

用于会员创建客服工单、查看工单列表、查看详情、回复工单和关闭工单。

入参概述：

会员身份、工单主题、内容、附件或工单上下文。

输出概述：

工单列表、工单详情、回复结果或关闭结果。

### 【接口：站内消息与公告】

调用方：前端请求。

功能说明：

用于首页公告、会员消息中心和站内消息展示。

入参概述：

会员身份或游客上下文、消息类型、分页条件。

输出概述：

公告内容、消息列表、阅读状态或详情。

### 【接口：聊天配置与 token】

调用方：前端请求 / 后端调用。

功能说明：

用于 Stream Chat 模式下获取聊天配置、生成用户 token 和创建频道。系统配置决定聊天是否启用、凭证是否完整和历史消息数量。

入参概述：

会员身份、聊天频道上下文和聊天配置。

输出概述：

聊天启用状态、客户端配置、聊天 token、频道结果或错误原因。

## 8. 代理接口

### 【接口：代理团队报表】

调用方：前端请求。

功能说明：

用于代理查看团队充值、提现、有效投注、盈亏和成员统计。

入参概述：

代理身份、时间范围、团队层级和筛选条件。

输出概述：

团队汇总、成员统计和分页结果。

### 【接口：代理下级管理】

调用方：前端请求。

功能说明：

用于代理新增成员、查看下级、设置代理关系或返点，以及查看下级详情。

入参概述：

代理身份、下级会员、账号信息、关系调整上下文。

输出概述：

成员列表、操作结果或失败原因。

### 【接口：代理团队充值】

调用方：前端请求。

功能说明：

用于代理给下级会员充值。系统要求幂等号，事务中扣减代理余额、增加下级余额、创建充值记录和两条转账流水。

入参概述：

代理身份、下级会员、充值金额、幂等业务号。

输出概述：

团队充值结果、订单状态、是否重复提交和余额变化。

### 【接口：代理邀请与佣金】

调用方：前端请求。

功能说明：

用于代理查看邀请码、邀请列表、佣金列表和返点信息。

入参概述：

代理身份、时间范围、分页或筛选条件。

输出概述：

邀请数据、佣金记录和返点信息。

## 9. 后台运营接口

### 【接口：后台资源 CRUD】

调用方：后台接口。

功能说明：

用于传统 Dcat Admin 资源管理，包括会员、充值、提现、游戏、活动、支付、代理、文章、日志等。

入参概述：

后台登录态、资源类型、筛选条件、表单数据和操作类型。

输出概述：

后台页面、保存结果、删除结果、审核结果或列表数据。

### 【接口：平台配置保存】

调用方：后台接口。

入参概述：

输出概述：

### 【接口：平台运营记录保存与导出】

入参概述：

后台用户身份、页面 code、筛选条件、记录数据、状态和导入导出上下文。

输出概述：

列表、保存结果、状态切换结果、删除结果、导入结果或导出内容。

### 【接口：游戏管理记录保存、状态、导入导出】

入参概述：

后台用户身份、游戏管理页面 code、字段数据、筛选条件、状态和导入导出上下文。

输出概述：

游戏管理列表、保存结果、状态变更、删除结果、导入结果或导出内容。

### 【接口：KYC 配置维护】

用于维护 KYC 字段、规则、前台内容和上传内容。

入参概述：

### 【接口：系统用户与权限维护】

用于后台系统用户、角色、权限、IP 白名单、任务和任务历史维护。

入参概述：

输出概述：

保存结果、权限结果、任务运行结果、历史记录或导出结果。

### 【接口：TCG shell 像素配置与操作日志】

入参概述：

输出概述：

### 【接口：后台实时客服会话处理】

调用方：后台接口。

功能说明：

用于后台客服人员查看打开中的会话、检索访客或会员、读取消息、发送客服回复、关闭会话和重新打开会话。后台读取消息会清理后台未读数，回复会增加用户侧未读数。

入参概述：

后台登录态、会话筛选条件、会话上下文和回复内容。

输出概述：

会话列表、消息列表、回复结果、关闭结果或重新打开结果。

## 11. 接口治理建议

### 推荐入口与兼容入口分级

当前系统存在新旧接口并存。建议为每类接口标注：

- 推荐入口：新前台、新 APP 和新后台使用。
- 兼容入口：旧客户端保留，不新增功能。
- 回调入口：第三方系统使用，必须验签和幂等。
- 待废弃入口：记录调用量后逐步迁移。

### 高风险接口清单

应重点测试和审计：

- 【接口：游戏余额转入转出】
- 【接口：WXGame 下注】
- 【接口：WXGame 派奖】
- 【接口：WXGame 退款】
- 【接口：充值申请】
- 【接口：支付回调处理】
- 【接口：提现申请】
- 【接口：代理团队充值】
- 【接口：活动申请】
- 【接口：实时客服消息发送】
- 【接口：后台实时客服会话处理】
- 【接口：平台配置保存】

### 统一鉴权建议

所有会员登录态接口应统一使用 API 鉴权中间件，减少控制器手动解析 token。所有后台写操作应统一通过 OperationPermission 能力断言。

## 12. 证据边界

已确认：

- 项目存在大量前台、后台、代理、第三方和内部任务接口。
- 前台接口语义覆盖用户、游戏、资金、活动、客服和代理。
- 后台接口语义覆盖传统 Dcat 资源和 TCG 页面契约。
- 内部实时客服具备前台会话、消息轮询、消息发送、会话关闭和后台接待语义。
- WXGame 回调具备独立的校验、余额、下注、派奖、退款和状态语义。

证据不足：

- 正式对外 API 文档版本。
- 所有旧 APP 接口的实际调用量。
- 第三方支付和游戏厂商的完整协议文档。
- 线上是否存在 API 网关、限流或 WAF。
